Particle Filter for SLAM: Dimensionality Problem 	[latex]Particle filters $=$ effective in low dimensional spaces\\$\Rightarrow$ likely regions of state space must be covered with samples!\\ \\Higher dimensions $=$ more samples\\$\Rightarrow x = (x_{1:t}, m_{1,x}, m_{1,y} \dots, m_{M,x}, m_{M,y})^\top$\\is fairly high-dimensional!\\ $\Rightarrow$ Particle filter application not straight-forward\\$\Rightarrow$ use rao-blackwellization to exploit dependencies![/latex]
Motivation+Intuition: Rao-Blackwellized Particle Filter  	Particle Filter for SLAM suffers from Dimensionality Problem=> Solution = use particle set only to model robot's path (each sample = path hypothesis)==> For each sample, compute individual map of landmarks===> Idea: Mapping given the poses is easy!
Rao-Blackwellization  	[latex]\begin{itemize}\item Factorization to exploit dependencies between variables:\begin{equation*}p(a,b) = p(b|a) p(a)\end{equation*}\item If $p(b|a)$ can be computed in closed form, represent only $p(a)$ withsamples and compute $p(b|a)$ for every sample\end{itemize}[/latex]
Modeling the Robot's Path in FASTSLAM  	[latex]\begin{itemize}\item Sample-based representation for\begin{equation*}p(x_{0:t}|z_{1:t}, u_{1:t})\end{equation*}\item Each sample $=$ path hypothesis\item Past poses of sample not revised\\$\Rightarrow$past poses not maintained in sample set\end{itemize}[/latex]

Key Steps of FastSLAM 1.0  	[latex]\begin{enumerate}\item Extend path posterior by sampling new pose for each sample:\begin{equation*}x^{[k]}_{t} \sim p(x_t | x^{[k]}_{t-1}, u_t)\end{equation*}\item Compute particle weight\begin{equation*}w^{[k]} = |2\pi Q|^{-\frac{1}{2}} exp \{-\frac{1}{2} (z_t -\underbrace{\tilde{z}^{[k]}}_{exp. obs})\top \underbrace{Q^{-1}}_{\text{obs. cov.}} (z_t - \tilde{z}^{[k]}) \}\end{equation*}\item Update belief of observed landmarks (EKF update)\item Resample\end{enumerate}[/latex]
